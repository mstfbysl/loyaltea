## Reporting a Vulnerability

We take the security of Loyaltea seriously. If you believe you have found a security vulnerability, please report it to us as described below.

### Please do NOT:

* Open a public GitHub issue for security vulnerabilities
* Discuss the vulnerability publicly until it has been addressed
* Exploit the vulnerability or access any data without permission

### How to Report

1. **Email the details** to: [security@mustafabaysal.dev](mailto:security@mustafabaysal.dev)

   Please include:
   * Type of issue (e.g., XSS, CSRF, SQL injection)
   * Full paths of source file(s) related to the manifestation of the issue
   * The location of the affected source code (tag/branch/commit or direct URL)
   * Step-by-step instructions to reproduce the issue
   * Proof-of-concept or exploit code (if possible)
   * Impact of the issue, including how an attacker might exploit the issue

2. **Wait for a response** - You should receive a response within 48 hours. If you don't, please follow up via email to ensure we received your report.

3. **Work with us** - Please allow us time to address the vulnerability before making any public disclosure.

### What to Expect

* **48 hours**: Initial response acknowledging receipt of your report
* **5 business days**: Update on the progress of addressing the vulnerability
* **Timeline for fix**: We'll work with you to understand the issue and develop a fix as quickly as possible

### Recognition

If you're the first to report a valid security vulnerability and we fix it, we would be happy to publicly acknowledge your responsible disclosure (if you wish). Please let us know how you would like to be credited.

## Security Best Practices

For users of Loyaltea:

* Keep your dependencies up to date
* Review and understand what code you're running
* Report security vulnerabilities responsibly
* Follow secure development practices when extending the project

## Scope

This security policy applies to:

* The Loyaltea codebase
* Dependencies that are part of the project
* Infrastructure used for hosting/deployment (if part of the repository)

Issues that are **out of scope**:

* Security issues in dependencies of Loyaltea that are unrelated to the project
* Issues requiring physical access to a user's machine
* Social engineering attacks
* Denial of service attacks
* Spam or content issues

Thank you for helping keep Loyaltea and its users safe!
